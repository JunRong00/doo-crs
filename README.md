# CRS Excel-to-XML Converter

Converts a filled CRS template Excel file into one or more **CRS XML Schema v3.0** files compliant with the OECD Amended CRS XML Schema User Guide v4.0 (October 2024).

---

## Prerequisites

- Python 3.8+
- `pandas`
- `openpyxl`

```bash
pip install pandas openpyxl
```

> **macOS with Anaconda:** Use `/opt/anaconda3/bin/python` to ensure both packages are available.

---

## Files

| File | Description |
|------|-------------|
| `xlsx_to_xml_converter.py` | Main converter — reads the Excel template, outputs CRS XML |
| `fill_dummy_data.py` | Fills `CRS_Template_Revised_2.xlsx` with sample data for testing |
| `CRS_Template_Revised_2.xlsx` | Blank CRS data entry template (6 sheets) |
| `CRS_Template_Dummy.xlsx` | Pre-filled sample template (generated by `fill_dummy_data.py`) |
| `CRS_Template_Dummy_CRS.xml` | Sample XML output generated from the dummy data |

---

## Template Structure

The Excel template has **6 sheets**:

| Sheet | Purpose |
|-------|---------|
| `MessageHeader` | Message-level metadata (sender, receiver, reporting period) |
| `ReportingFI` | Reporting Financial Institution details |
| `Individual` | Individual account holder records |
| `Organisation` | Entity/organisation account holder records |
| `ControllingPerson` | Controlling persons linked to organisation accounts |
| `Payment` | Payment records linked to individual or organisation accounts |

**Layout convention:**
- Row 1: Column headers
- Rows 2–6: Metadata / remarks (auto-skipped by converter)
- Row 7: Blank separator
- Row 8+: Data records

## Usage

### Convert template to XML

```bash
python xlsx_to_xml_converter.py CRS_Template_Revised_2.xlsx
```

Output: `CRS_Template_Revised_2_CRS.xml` + `CRS_Template_Revised_2_with_filepath.xlsx`

### Split into multiple XML files

```bash
python xlsx_to_xml_converter.py CRS_Template_Revised_2.xlsx --split 3
```

Output: `..._CRS_part01.xml`, `..._CRS_part02.xml`, `..._CRS_part03.xml`

### Custom output prefix

```bash
python xlsx_to_xml_converter.py CRS_Template_Revised_2.xlsx --out submissions/MY2024
```

### Full options

```
usage: xlsx_to_xml_converter.py [-h] [--split N] [--out PREFIX] input

Arguments:
  input           Path to the filled CRS Excel template (.xlsx)
  --split N, -s N Split output into N XML files (default: 1)
  --out PREFIX    Output filename prefix (default: input filename stem)
```

---

## Outputs

1. **XML file(s)** — One or more CRS XML Schema v3.0 files ready for submission.
2. **Excel with filepath column** — A copy of the input Excel with a `filepath` column added to `Individual`, `Organisation`, `ControllingPerson`, and `Payment` sheets, indicating which XML file each row was written to.

---

## Splitting Logic

- Splitting unit = account rows (`Individual` + `Organisation` combined).
- Rows are distributed using ceiling division — no file has more than one extra row compared to any other.
- Each split file receives a unique `MessageRefId`.
- The `ReportingFI` block is replicated in every split file (required by schema).
- `ControllingPerson` and `Payment` rows travel with their parent account.

### Nil Return

Set `messageTypeIndic` to `CRS703` in `MessageHeader` to generate a nil-return XML (no `CrsBody`).

---

## Key Schema Notes

- **SelfCert for AccountHolder** uses `CrsSelfCert_EnumType`: `CRS901` (true), `CRS902` (false), `CRS900` (not reported).
- **SelfCert for ControllingPerson** uses `CrsSelfCertforCtrlgPerson_EnumType`: `CRS1001` (true), `CRS1002` (false), `CRS1000` (not reported).
- `CorrMessageRefId` is **not used** in CRS — it is marked "Optional (non-CRS)" in the OECD spec (p.9, p.26). Use `corrDocRefId` for corrections.
- Reference: [OECD CRS XML Schema User Guide v4.0](https://doi.org/10.1787/dd7ee57a-en)
